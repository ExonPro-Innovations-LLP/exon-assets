Common Headers

All requests use HTTPS (TLS 1.2+). All responses are JSON (Content-Type: application/json).

HeaderValueDescription
AuthorizationBearer {MASKED_JWT_TOKEN}RS256 JWT issued by xVio Scan (required on authenticated endpoints)
X-App-IDxvio-scanApplication identifier (required on tenant endpoints)
Content-Typeapplication/jsonRequest body format
Originhttps://scan.xvio.aiCORS origin (validated against allowlist)

Error Response Format (all endpoints)

HTTP/1.1 {statusCode} Content-Type: application/json { "error": "ERROR_CODE", "message": "Human-readable description", "errorCode": "ERROR_CODE" }

Common error codes: UNAUTHORIZED, FORBIDDEN, NOT_FOUND, VALIDATION_ERROR, RATE_LIMITED, QUOTA_EXCEEDED

1. Authentication Endpoints

1a. POST /tenant/auth/login
Initiates Salesforce OAuth 2.0 + PKCE login flow. Returns available auth methods and Salesforce authorization URL.
REQUEST
POST /v1/tenant/auth/login HTTP/1.1 Host: api.xvio.ai Content-Type: application/json X-App-ID: xvio-scan Origin: https://scan.xvio.ai { "customId": "free", "redirectUri": "https://scan.xvio.ai/auth/callback" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json Access-Control-Allow-Origin: https://scan.xvio.ai Access-Control-Allow-Credentials: true { "authMethods": { "salesforce": { "authUrl": "https://login.salesforce.com/services/oauth2/authorize?response_type=code&client_id={SF_CLIENT_ID}&redirect_uri=...&state={CSRF_STATE}&code_challenge={PKCE_CHALLENGE}&code_challenge_method=S256&scope=api+refresh_token", "state": "{CSRF_STATE}" }, "cognito": { "enabled": true } }, "tenant": { "id": "{TENANT_ID}", "name": "xVio Free", "customId": "free" } }
Rate limit: 10 requests per minute per IP.
1b. GET /tenant/auth/callback
Salesforce OAuth callback. Exchanges authorization code for tokens. Redirects to web app or mobile deep link.
REQUEST
GET /v1/tenant/auth/callback?code={SF_AUTH_CODE}&state={CSRF_STATE} HTTP/1.1 Host: api.xvio.ai
RESPONSE 302 (Web)
HTTP/1.1 302 Found Location: https://scan.xvio.ai/auth/callback?authSuccess=true Set-Cookie: token={MASKED_JWT_TOKEN}; HttpOnly; Secure; SameSite=None; Domain=.xvio.ai; Path=/; Max-Age=900 Set-Cookie: tenantId={TENANT_ID}; Secure; SameSite=None; Domain=.xvio.ai; Path=/; Max-Age=2592000
RESPONSE 302 (Mobile)
HTTP/1.1 302 Found Location: xvio://oauth/callback?authSuccess=true&code={ONE_TIME_CODE}&tenantId={TENANT_ID}
Mobile redirects use a one-time auth code (60s TTL) instead of tokens in the URL.
1c. POST /tenant/auth/exchange
Exchanges one-time auth code for JWT tokens. Used by mobile apps after deep link callback.
REQUEST
POST /v1/tenant/auth/exchange HTTP/1.1 Host: api.xvio.ai Content-Type: application/json { "code": "{ONE_TIME_AUTH_CODE}" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json { "accessToken": "{MASKED_JWT_TOKEN}", "token": "{MASKED_JWT_TOKEN}", "refreshToken": "{REFRESH_TOKEN}", "expiresIn": 900, "tenantId": "{TENANT_ID}", "user": { "id": "{USER_ID}", "email": "{USER_EMAIL}", "name": "{USER_NAME}" } }
The one-time code expires after 60 seconds and is deleted after first use.
1d. POST /tenant/auth/refresh
Refreshes an expired JWT using the refresh token.
REQUEST
POST /v1/tenant/auth/refresh HTTP/1.1 Host: api.xvio.ai Content-Type: application/json { "refreshToken": "{REFRESH_TOKEN}", "tenantId": "{TENANT_ID}" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json Set-Cookie: token={MASKED_JWT_TOKEN}; HttpOnly; Secure; SameSite=None; Domain=.xvio.ai; Path=/; Max-Age=900 { "accessToken": "{MASKED_JWT_TOKEN}", "expiresIn": 900 }
Rate limit: 30 requests per minute per IP.
1e. POST /tenant/auth/logout
Invalidates session, revokes JWT and optionally the refresh token.
REQUEST
POST /v1/tenant/auth/logout HTTP/1.1 Host: api.xvio.ai Authorization: Bearer {MASKED_JWT_TOKEN} Content-Type: application/json { "refreshToken": "{REFRESH_TOKEN}" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json Set-Cookie: token=; HttpOnly; Secure; SameSite=None; Domain=.xvio.ai; Path=/; Max-Age=0 Set-Cookie: tenantId=; Secure; SameSite=None; Domain=.xvio.ai; Path=/; Max-Age=0 { "success": true, "message": "Logged out successfully" }

2. Scanning and Jobs Endpoints

2a. POST /tenant/jobs
Upload a document for AI scanning. Accepts base64-encoded file.
REQUEST
POST /v1/tenant/jobs HTTP/1.1 Host: api.xvio.ai Authorization: Bearer {MASKED_JWT_TOKEN} Content-Type: application/json X-App-ID: xvio-scan { "tenantId": "{TENANT_ID}", "userId": "{USER_ID}", "serviceId": "{SERVICE_ID}", "file": { "name": "business-card.jpg", "type": "image/jpeg", "data": "{BASE64_FILE_DATA}" } }
RESPONSE 201 Created
HTTP/1.1 201 Created Content-Type: application/json { "jobId": "{JOB_ID}", "status": "pending", "progress": 0, "serviceId": "{SERVICE_ID}", "serviceName": "Business Card Scanner", "createdAt": "2026-03-09T10:00:00.000Z" }
File size limit: 20MB for base64. For larger files (up to 50MB), use the presigned upload endpoint.
2b. POST /tenant/services/{serviceId}/upload-presigned
Get a presigned S3 URL for large file uploads.
REQUEST
POST /v1/tenant/services/{SERVICE_ID}/upload-presigned HTTP/1.1 Host: api.xvio.ai Authorization: Bearer {MASKED_JWT_TOKEN} Content-Type: application/json X-App-ID: xvio-scan { "tenantId": "{TENANT_ID}", "userId": "{USER_ID}", "filename": "invoice-scan.pdf", "contentType": "application/pdf" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json { "uploadUrl": "{PRESIGNED_S3_URL}", "jobId": "{JOB_ID}", "expiresIn": 300 }

3. Salesforce Export Endpoints

3a. POST /tenant/ai/exporters/salesforce
Export scan data to Salesforce CRM as a Lead, Contact, Case, or custom sObject.
REQUEST
POST /v1/tenant/ai/exporters/salesforce HTTP/1.1 Host: api.xvio.ai Authorization: Bearer {MASKED_JWT_TOKEN} Content-Type: application/json X-App-ID: xvio-scan { "tenantId": "{TENANT_ID}", "userId": "{USER_ID}", "jobId": "{JOB_ID}", "createdAt": "2026-03-09T10:00:00.000Z", "objectType": "Lead" }
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json { "exportId": "{EXPORT_ID}", "sfRecordId": "{SF_RECORD_ID}", "message": "Exported successfully" }
Uses admin-configured field mappings. Backend authenticates to Salesforce using stored OAuth tokens (AES-256-GCM encrypted). Auto-refreshes expired tokens.

View export history for the current user.
REQUEST
GET /v1/tenant/ai/exporters/history?tenantId={TENANT_ID}&userId={USER_ID} HTTP/1.1 Host: api.xvio.ai Authorization: Bearer {MASKED_JWT_TOKEN} X-App-ID: xvio-scan
RESPONSE 200 OK
HTTP/1.1 200 OK Content-Type: application/json { "exports": [ { "exportId": "{EXPORT_ID}", "jobId": "{JOB_ID}", "destination": "salesforce", "objectType": "Lead", "sfRecordId": "{SF_RECORD_ID}", "status": "success", "exportedAt": "2026-03-09T10:10:00.000Z" } ], "total": 1 }

5. JWT Token Structure

All authenticated endpoints require a Bearer JWT token. The JWT payload contains:

{ "sub": "{USER_SUB}", "email": "{USER_EMAIL}", "tenantId": "{TENANT_ID}", "userId": "{USER_ID}", "type": "tenant_user", "jti": "{UNIQUE_JWT_ID}", "iss": "api.xvio.ai", "aud": "xvio-api", "iat": 1709971200, "exp": 1709972100 }
SettingValue
AlgorithmRS256 (asymmetric)
Default expiry15 minutes (configurable per tenant: 5 to 1440 minutes)
Signing keyStored in AWS Secrets Manager, cached in Lambda for 5 minutes

6. CORS and Security Headers

All API responses include the following security headers (via CloudFront Function):

Strict-Transport-Security: max-age=63072000; includeSubDomains; preload X-Content-Type-Options: nosniff X-Frame-Options: DENY X-XSS-Protection: 1; mode=block Referrer-Policy: strict-origin-when-cross-origin Permissions-Policy: camera=(), microphone=(), geolocation=()

CORS headers on authenticated responses:

Access-Control-Allow-Origin: https://scan.xvio.ai Access-Control-Allow-Credentials: true Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS Access-Control-Allow-Headers: Authorization, Content-Type, X-App-ID, X-API-Key, X-API-Secret
