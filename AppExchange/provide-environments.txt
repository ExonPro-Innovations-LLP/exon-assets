OTHER INFORMATION
-----------------
Testing Instructions:

1. AUTHENTICATION FLOW:
   - Navigate to https://scan.xvio.ai
   - Click "Login with Salesforce"
   - Enter the Salesforce test org credentials provided above
   - You will be redirected back to xVio Scan, authenticated

2. DOCUMENT SCANNING:
   - Click "New Scan" and select a document type (Business Card, Invoice, Expense, ID Document)
   - Upload a sample document image or use camera capture
   - The AI pipeline processes the document: Textract OCR → Bedrock LLM field extraction
   - Review extracted fields and edit if needed

3. SALESFORCE EXPORT:
   - After scanning, click "Export to Salesforce"
   - The extracted data is sent to Salesforce via REST API
   - Verify the record was created in the Salesforce test org (Leads, Contacts, or Cases depending on document type)

4. IMPORTANT NOTES:
   - xVio Scan does NOT install any code in the Salesforce org (no managed package, no Apex, no Lightning Components)
   - The only Salesforce integration is through the Connected App OAuth flow and Salesforce REST API
   - All processing happens on AWS infrastructure (Lambda, DynamoDB, S3, Textract, Bedrock)
   - No Salesforce data is persisted outside Salesforce except OAuth tokens (encrypted with AES-256-GCM in AWS Secrets Manager) and basic user profile (email, name) for session display
   - The test tenant has quota limits configured — if you hit a quota limit, contact support@xvio.ai

5. SECURITY CONTROLS ACTIVE IN PRODUCTION:
   - AWS WAF v2 with OWASP rules (may block some scanner traffic — contact us if needed)
   - Rate limiting on auth endpoints (DynamoDB-based per-IP counters)
   - API Gateway throttling (1000 req/min, 500 burst)
   - CORS with explicit origin allowlist
   - TLS 1.2+ enforced on all connections

6. SUPPORT CONTACT:
   - Email: support@xvio.ai
   - If you encounter any issues during testing, please reach out and we will assist promptly.
