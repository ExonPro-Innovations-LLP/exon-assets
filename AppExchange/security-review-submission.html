1. Solution Overview

xVio Scan is an AI-powered document scanning platform that transforms physical documents — business cards, invoices, receipts, ID documents, forms, and contracts — into structured, exportable data. It integrates natively with Salesforce as a Connected App, enabling organisations to capture document data directly into Salesforce CRM objects.

How It Works

1
Capture — Users scan documents via mobile camera (iOS/Android) or upload images/PDFs from the web interface.
2
OCR Extraction — AWS Textract extracts raw text, layout, tables, and form fields from the document image.
3
AI Field Extraction — AWS Bedrock (LLM) parses the extracted text into structured fields (names, emails, phone numbers, addresses, invoice line items, etc.) using a configurable prompt template per document type.
4
Auto Export — Extracted data is automatically exported to Salesforce as Leads, Contacts, Cases, or custom objects with admin-configurable field mappings.

4. Data Flow — Scan to Salesforce

1
Upload — Client sends document image (JPEG/PNG/PDF/TIFF, max 50MB) via POST /tenant/jobs or presigned S3 upload. File stored in S3 under {tenantId}/{appId}/{jobId}/ prefix with SSE-S3 encryption.
2
Queue — Upload handler validates quota (dual-level: per-user + per-org), creates a DynamoDB job record, and pushes a message to the SQS Jobs queue.
3
OCR — The AI orchestrator Lambda consumes from SQS. Stage 1 calls AWS Textract (DetectDocumentText, AnalyzeDocument, AnalyzeExpense, or AnalyzeID depending on the service pipeline config).
4
AI Extraction — Stage 2 sends OCR text to AWS Bedrock via the Converse API. The LLM extracts structured fields based on a per-service prompt template. Output is validated against the service's field schema.
5
Store — Extracted data saved to DynamoDB. Real-time status update pushed to client via AppSync WebSocket subscription.
6
Export — User triggers export. The exporter Lambda authenticates to Salesforce using stored (encrypted) OAuth tokens, maps fields per admin config, and creates/updates the Salesforce record via REST API.
7
Cleanup — If deleteSourceAfterProcessing is enabled, the source image is deleted from S3 after successful processing.

In TransitTLS 1.2+CloudFront enforces HTTPS. API Gateway HTTPS-only origin policy.

S3 at RestSSE-S3 (AES-256)blockPublicAccess: BLOCK_ALL on all buckets.

Data Handling Policies

  • No PII in logs — Structured JSON logging with tenantId, userId, action, correlationId. No emails, names, or document content logged.
  • Source file deletion — Configurable deleteSourceAfterProcessing flag per service. Diagnostic log retention: 7 days (dev), 365 days (prod).
  • API response masking — Salesforce credentials masked in API responses (e.g., 3MVG9...***). Tokens never returned in list/get endpoints.
  • Logout cleanup — clearAllData() removes auth tokens, IndexedDB, localStorage, and sync cache from client.

8. Multi-Tenant Isolation

LayerIsolation Mechanism
DynamoDBPartition key always includes TENANT#{tenantId}#APP#{appId}. Cross-tenant queries are structurally impossible — not enforced by application logic, but by data architecture.
S3Key prefix includes {tenantId}/. Access scoped per request.
JWT ClaimstenantId extracted from JWT token, never from request parameters. Cannot be spoofed without the signing key.
Secrets ManagerPer-tenant credential paths: /xvio/{stage}/tenant/{tenantId}/sf-oauth
Multi-AppX-App-ID header required. PK includes APP#{appId}.
